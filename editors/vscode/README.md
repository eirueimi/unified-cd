# unified-cd YAML (VS Code Extension)

A VS Code extension that provides schema completion, validation, hover, and cross-reference
completion for unified-cd YAML resources (`apiVersion: unified-cd/v1`).

## Features

- **Schema completion (zero config)**: Open any YAML file containing `apiVersion: unified-cd/v1`
  and get key completion, enum completion, validation, and hover out of the box.
  No settings or modelines required. Completion is delegated to `redhat.vscode-yaml`.
- **Cross-reference completion**: Candidates are offered for fields that reference other resources
  in the workspace:
  - `Schedule` `spec.job` → names of `kind: Job` resources in the workspace
  - `WebhookReceiver` `spec.trigger.job` → same as above
  - `AppSource` `spec.gitCredentialRef` → names of `kind: GitCredential` resources in the workspace

## Limitations (current version)

- `secretRef` (StoredSecret) and `agentSelector` values are not completion candidates.
- Candidates are derived by scanning YAML files in the workspace (no live server connection).
- `job:` value completion is triggered by the field name (`job:` at the start of the line),
  so Job name candidates may appear for `job:` fields that do not reference a registered Job
  (e.g. the `job:` field in a `uses` step specifying a git URI).

## Build and install

The `redhat.vscode-yaml` dependency is installed automatically by VS Code when you install the VSIX.

```bash
# From the repository root
make vscode-package
# -> generates editors/vscode/unified-cd-yaml.vsix

# Install (CLI)
code --install-extension editors/vscode/unified-cd-yaml.vsix
```

To install via the GUI: open the Extensions view, click the `…` menu, choose
**Install from VSIX...**, and select the generated `.vsix` file.

## Usage

1. Open a unified-cd YAML file (e.g. `examples/*.yaml`) in VS Code.
2. Type a key or enum value to get completion and validation.
3. Trigger completion at the value position of `job:` or `gitCredentialRef:` to see
   matching resource names from the workspace.

## Development

```bash
cd editors/vscode
npm install
npm test          # vitest (unit tests for pure logic)
npm run build     # copy schema and bundle with esbuild
# Press F5 in VS Code to launch an Extension Development Host for manual testing
```

The schema is sourced from the single source of truth `schemas/unified-cd.schema.json`
(generated by `cmd/schemagen`) and copied into `schema/` via `npm run copy-schema`.
Do not hand-edit `schema/unified-cd.schema.json`.
